# 인형뽑기 - Stack

##  문제 요약
- board 2차원 배열에 인형이 쌓여있음
- moves 배열 순서대로 열을 선택해서 인형을 뽑음
- 뽑은 인형을 스택에 쌓다가 연속으로 같은 인형이 오면 터뜨림
- 터진 인형의 총 개수를 반환

---

##  왜 Stack을 사용했는가?

- 가장 최근에 뽑은 인형과 비교해야 하기 때문
- **마지막에 넣은 값을 가장 먼저 꺼내는 LIFO 구조**가 딱 맞음
- 배열로도 가능하지만 Stack을 쓰면 peek()으로 top 바로 확인 가능

---

##  핵심 알고리즘

### 1. 열 선택 후 위에서부터 순회
```
moves 순서대로 열(col) 선택
→ board[row][col] 을 row 0부터 내려오며 순회
→ 0이 아닌 첫 번째 값이 인형
→ 뽑은 자리는 0으로 변경
```

### 2. 스택에 넣고 비교
```
뽑은 인형을 스택 top과 비교
→ 같으면? → pop + count += 2 
→ 다르면? → push
```


---

## 최종 코드
```java
import java.util.*;

class Solution {
    public int solution(int[][] board, int[] moves) {
        int count = 0;
        Stack<Integer> stack = new Stack<>();

        for (int move : moves) {
            int col = move - 1; // moves는 1부터 시작이라 -1

            for (int row = 0; row < board.length; row++) {
                if (board[row][col] != 0) {
                    int doll = board[row][col];
                    board[row][col] = 0; // 뽑은 자리 0으로

                    if (!stack.isEmpty() && stack.peek() == doll) {
                        stack.pop();
                        count += 2;
                    } else {
                        stack.push(doll);
                    }
                    break; // 인형 하나만 뽑고 다음 move로
                }
            }
        }
        return count;
    }
}
```

## 느낀 점
- 처음엔 어떤 자료구조를 사용할지에 대해 고민을 계속했다. 사실 Stack을 이용해서 문제를 푼 적은 처음이다. 그래서 push랑 pop 문법을 배우면서 풀었다.
- board[row][col] 에서 col을 고정하고 row를 0부터 내려가며 순회하는 구조가 처음엔 헷갈렸는데, 크레인이 위에서 아래로 내려오며 첫 번째 인형을 집는 거라고 생각하니 이해됐다.
- 앞으로 이런 문제가 나오면 LIFO 구조를 떠올리며 바로 Stack과 연관지어야겠다.
