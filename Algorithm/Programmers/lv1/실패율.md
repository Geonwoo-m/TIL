# 프로그래머스 - 실패율

- **분류**: 구현 / 정렬
- **플랫폼**: 프로그래머스
- **링크**: https://school.programmers.co.kr/learn/courses/30/lessons/42889

---

## 문제 요약

- N개의 스테이지가 있고, `stages` 배열에는 각 플레이어가 현재 도전 중인 스테이지 번호가 담겨 있음
- 실패율 = 스테이지에 **도달했으나 클리어하지 못한 플레이어 수** / 스테이지에 **도달한 플레이어 수**
- 실패율이 높은 스테이지부터 내림차순 정렬해서 반환
- 실패율이 같으면 스테이지 번호가 작은 것이 앞에 옴

---


## 핵심 로직

1. `reach` 변수를 `stages.length`로 초기화 → 1스테이지에 도달한 사람 = 전체 플레이어
2. 스테이지 1번부터 N번까지 순회하면서:
   - 해당 스테이지에 **머물고 있는 플레이어 수** (`count`) 계산
   - 실패율 = `count / reach` (도달자가 0이면 실패율 0으로 처리)
   - `reach -= count` 로 다음 스테이지 도달자 수 갱신
3. 스테이지 번호 배열을 실패율 기준 **내림차순** 정렬 (같으면 스테이지 번호 오름차순)
4. `Integer[]` → `int[]` 변환 후 반환

---

## 정렬 람다 이해

```java
Arrays.sort(stageNums, (a, b) -> {
    if (failRate[a] != failRate[b]) return Double.compare(failRate[b], failRate[a]);
    return a - b;
});
```

- `failRate[b] - failRate[a]` 방식은 `double` 비교 시 정밀도 오류가 생길 수 있어 `Double.compare()` 사용
- 실패율이 같으면 `a - b`로 스테이지 번호 오름차순 유지

---

## 최종 코드

```java
import java.util.*;

class Solution {
    public int[] solution(int N, int[] stages) {
        double[] failRate = new double[N + 2];
        int reach = stages.length;

        for (int i = 1; i <= N; i++) {
            int count = 0;
            for (int s : stages) {
                if (s == i) count++;
            }
            if (reach == 0) {
                failRate[i] = 0;
            } else {
                failRate[i] = (double) count / reach;
            }
            reach -= count;
        }

        Integer[] stageNums = new Integer[N];
        for (int i = 0; i < N; i++) stageNums[i] = i + 1;

        Arrays.sort(stageNums, (a, b) -> {
            if (failRate[a] != failRate[b]) return Double.compare(failRate[b], failRate[a]);
            return a - b;
        });

        int[] result = new int[N];
        for (int i = 0; i < N; i++) result[i] = stageNums[i];
        return result;
    }
}
```

---

## 느낀 점

- `reach`를 매 스테이지마다 줄여가는 방식이 핵심 — 이전 스테이지에 머문 사람 수를 빼면 다음 스테이지 도달자 수를 별도 계산 없이 구할 수 있음
- `double` 비교 시 `==` 대신 `Double.compare()`를 써야 부동소수점 오차를 피할 수 있음
- 정렬 기준이 2개(실패율 내림차순, 스테이지 번호 오름차순)일 때 람다 안에서 조건 분기하는 패턴을 익힘
- `int[]`는 람다에서 직접 정렬 불가 → `Integer[]`로 박싱 후 정렬, 마지막에 언박싱하는 흐름을 기억할 것
