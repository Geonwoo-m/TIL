# 완주하지 못한 선수 - HashMap

##  문제 요약
- participant 배열에 모든 선수 이름이 있음
- completion 배열에 완주한 선수 이름이 있음
- 완주하지 못한 선수 1명을 찾아서 반환
- 동명이인이 있을 수 있음

---

##  왜 HashMap을 사용했는가?

- 이름(String) → 등장 횟수(int) 매핑이 필요한 구조
- 동명이인 처리를 위해 횟수를 카운팅해야 함
- 배열로 하면 동명이인 처리가 복잡해짐
- HashMap을 쓰면 이름으로 바로 접근 가능 → O(1)

---

##  핵심 알고리즘 

### 1단계 - participant 전원 HashMap에 등록
```
모든 참가자 이름을 key로, 등장 횟수를 value로 저장
동명이인이면 +1 카운팅
```

### 2단계 - completion 기준으로 횟수 차감
```
완주한 선수 이름이 나올 때마다 횟수 -1
```

### 3단계 - value가 1 이상인 이름 반환
```
완주 못한 선수는 차감 후에도 횟수가 남아있음
```

---

##  예시 
```
participant  = ["mislav", "stanko", "mislav", "ana"]
completion   = ["stanko", "ana", "mislav"]

1단계 HashMap 등록
mislav → 2
stanko → 1
ana    → 1

2단계 completion 차감
stanko → 0
ana    → 0
mislav → 1

3단계 value >= 1 인 key 반환
→ mislav ✅
```

---

## 최종 코드
```java
import java.util.*;

class Solution {
    public String solution(String[] participant, String[] completion) {
        Map<String, Integer> map = new HashMap<>();

        // 1단계: 참가자 전원 등록
        for (String p : participant) {
            map.put(p, map.getOrDefault(p, 0) + 1);
        }

        // 2단계: 완주자 차감
        for (String c : completion) {
            map.put(c, map.get(c) - 1);
        }

        // 3단계: 완주 못한 선수 반환
        for (String key : map.keySet()) {
            if (map.get(key) >= 1) {
                return key;
            }
        }
        return "";
    }
}
```

---

## 느낀 점
- 동명이인 처리 때문에 단순 비교로는 풀 수 없고 횟수 카운팅이 필요했음
- getOrDefault() 를 쓰면 key가 없을 때 기본값을 설정할 수 있어서 편함
- 앞으로 "특정 값의 등장 횟수"가 필요한 문제에서는 HashMap을 먼저 떠올려야겠다
