# 키패드 누르기 - 2차원 배열 + 맨해튼 거리
## 📌 문제 요약
- 키패드에서 numbers 배열 순서대로 번호를 누름
- 1,4,7 → 무조건 왼손 / 3,6,9 → 무조건 오른손
- 2,5,8,0 → 왼손/오른손 중 더 가까운 손 사용
- 거리가 같으면 주손 사용

---

## 키패드를 격자로 표현
```
열:  0    1    2
행0 [1]  [2]  [3]
행1 [4]  [5]  [6]
행2 [7]  [8]  [9]
행3 [*]  [0]  [#]
```

각 숫자의 위치를 {행, 열} 로 미리 저장
```java
private static final int[][] POSITIONS = {
    {3, 1}, // 0 → 3행 1열
    {0, 0}, // 1 → 0행 0열
    {0, 1}, // 2 → 0행 1열
    {0, 2}, // 3 → 0행 2열
    {1, 0}, // 4 → 1행 0열
    {1, 1}, // 5 → 1행 1열
    {1, 2}, // 6 → 1행 2열
    {2, 0}, // 7 → 2행 0열
    {2, 1}, // 8 → 2행 1열
    {2, 2}  // 9 → 2행 2열
};
// POSITIONS[5] → {1, 1} (5번 키의 위치)
```

## 💡 맨해튼 거리란?

대각선 없이 **상하좌우로만 이동할 때의 거리**

예) 1번(0행 0열) → 5번(1행 1열)
```
아래로 1칸 + 오른쪽 1칸 = 2
```
```java
private int getDistance(int[] current, int[] target) {
    return Math.abs(current[0] - target[0])  // 행 차이
         + Math.abs(current[1] - target[1]); // 열 차이
}
// Math.abs() → 음수 방지를 위해 절댓값 처리
```

## 🔑 핵심 알고리즘

### 1단계 - 초기 손가락 위치 설정
```
왼손 → * 위치 {3, 0}
오른손 → # 위치 {3, 2}
```

### 2단계 - 번호별 손 결정
```
1, 4, 7 → 무조건 왼손 (왼쪽 열)
3, 6, 9 → 무조건 오른손 (오른쪽 열)
2, 5, 8, 0 → 거리 계산 후 가까운 손 사용
            → 동일하면 주손(hand) 사용
```

### 3단계 - 손가락 위치 업데이트
```
누른 손가락 위치를 POSITIONS[num] 으로 갱신
```

## 최종 코드
```java
class Solution {
    private static final int[][] POSITIONS = {
        {3, 1}, // 0
        {0, 0}, {0, 1}, {0, 2}, // 1, 2, 3
        {1, 0}, {1, 1}, {1, 2}, // 4, 5, 6
        {2, 0}, {2, 1}, {2, 2}  // 7, 8, 9
    };

    public String solution(int[] numbers, String hand) {
        StringBuilder sb = new StringBuilder();
        int[] left = {3, 0};  // * 위치
        int[] right = {3, 2}; // # 위치

        for (int num : numbers) {
            if (num == 1 || num == 4 || num == 7) {
                sb.append("L");
                left = POSITIONS[num];
            } else if (num == 3 || num == 6 || num == 9) {
                sb.append("R");
                right = POSITIONS[num];
            } else {
                int leftDist = getDistance(left, POSITIONS[num]);
                int rightDist = getDistance(right, POSITIONS[num]);
                if (leftDist < rightDist) {
                    sb.append("L");
                    left = POSITIONS[num];
                } else if (rightDist < leftDist) {
                    sb.append("R");
                    right = POSITIONS[num];
                } else {
                    if (hand.equals("left")) {
                        sb.append("L");
                        left = POSITIONS[num];
                    } else {
                        sb.append("R");
                        right = POSITIONS[num];
                    }
                }
            }
        }
        return sb.toString();
    }

    private int getDistance(int[] current, int[] target) {
        return Math.abs(current[0] - target[0]) + Math.abs(current[1] - target[1]);
    }
}
```

## 느낀 점
- 솔직히 처음엔 어떻게 풀어야할지 감도 안잡혔다. 그러다가 AI에게 코드는 보내주지말고 어떤식으로 풀어야하는지 유도해달라고 했는데, 키패드를 좌표값으로 설정하고 L이나 R을 계속 붙여나가면 된다고 했다.
- 처음엔 듣고 StringBuilder까지는 알겠는데 좌표값으로 어떻게 설정하지에 대한 고민을 했다가 2차원 배열 생각이 나서 그렇게 하게 되었다.
- POSITIONS를 static final 상수로 선언한 이유는 키패드 좌표는 절대 바뀌지 않는 값이기 때문에 상수로 선언해 불필요한 메모리 낭비를 줄였다.
- 맨해튼 거리 공식도 처음 알게 되었는데, 앞으로 격자에서 두 점 사이의 거리를 구해야 할 때 바로 떠올려야겠다.
