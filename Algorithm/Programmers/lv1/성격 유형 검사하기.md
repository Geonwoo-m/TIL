# 성격 유형 검사하기 - HashMap

## 문제 요약
- survey 배열의 각 질문에서 선택지(choices)에 따라 점수를 부여
- R-T, C-F, J-M, A-N 4쌍을 비교해서 높은 점수의 유형을 반환
- 동점이면 알파벳 앞글자(R, C, J, A) 선택

---

##  왜 HashMap을 사용했는가?

- 8개의 유형(R,T,C,F,J,M,A,N) 각각에 점수를 저장해야 함
- `문자(char) → 점수(int)` 매핑이 필요한 구조
- 배열로도 가능하지만 HashMap을 쓰면 문자를 key로 직접 접근 가능

---

##  핵심 알고리즘 3단계

### 1단계 - 8가지 유형 점수 0으로 초기화
```java
for (char c : new char[]{'R','T','C','F','J','M','A','N'}) {
    test.put(c, 0);
}
```

### 2단계 - 선택지에 따라 점수 부여
```
choices[i] < 4 → 앞 글자(first)에 (4 - choices[i]) 점수 추가
choices[i] > 4 → 뒷 글자(second)에 (choices[i] - 4) 점수 추가
choices[i] == 4 → 중립, 점수 없음
```

### 3단계 - 4쌍 비교 후 결과 문자열 생성
```
R vs T, C vs F, J vs M, A vs N 순서로 비교
→ 점수 높은 쪽 선택
→ 동점이면 앞 글자(R, C, J, A) 선택
```

---

## 최종 코드
```java
import java.util.*;
import java.util.Map;

class Solution {
    public String solution(String[] survey, int[] choices) {
        Map<Character, Integer> test = new HashMap<>();
        for (char c : new char[]{'R','T','C','F','J','M','A','N'}) {
            test.put(c, 0);
        }
        for (int i = 0; i < survey.length; i++) {
            char first = survey[i].charAt(0);
            char second = survey[i].charAt(1);
            if (choices[i] < 4) {
                test.put(first, test.get(first) + (4 - choices[i]));
            } else if (choices[i] > 4) {
                test.put(second, test.get(second) + (choices[i] - 4));
            }
        }
        StringBuilder answer = new StringBuilder();
        String[][] pairs = {{"R","T"}, {"C","F"}, {"J","M"}, {"A","N"}};
        for (String[] pair : pairs) {
            char a = pair[0].charAt(0);
            char b = pair[1].charAt(0);
            answer.append(test.get(a) >= test.get(b) ? pair[0] : pair[1]);
        }
        return answer.toString();
    }
}
```

---

## 느낀 점
- 처음에 배열로 하려다가, HashMap을 사용하면 문자로 직접 접근이 가능해서 HashMap을 쓰게되었다.
- 앞으로 `문자 → 값` 매핑이 필요한 문제에서는 HashMap을 먼저 떠올릴 것 같다.
