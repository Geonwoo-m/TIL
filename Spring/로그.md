# Spring 로그(Log) 기초

## 왜 System.out.println() 대신 로그를 쓰는가?

| System.out.println() | Log |
|------|------|
| 부가 정보 없음 | 시간, 클래스명, 쓰레드 정보 등 포함 |
| 레벨 조절 불가 | 개발/운영 서버별 레벨 조절 가능 |
| 콘솔에만 출력 | 파일, 네트워크 등 별도 저장 가능 |
| 성능 느림 | 내부 버퍼링으로 성능 우수 |

---

##  로깅 라이브러리 구조
```
SLF4J (인터페이스)
└── Logback (구현체) ← 스프링 부트 기본값
└── Log4J
└── Log4J2
```
> SLF4J는 인터페이스, Logback은 그 구현체. 실무에서는 Logback을 주로 사용

##  로그 선언 방법 3가지
```java
// 방법 1
private Logger log = LoggerFactory.getLogger(getClass());

// 방법 2
private static final Logger log = LoggerFactory.getLogger(Xxx.class);

// 방법 3 - 롬복 사용 (가장 간편)
@Slf4j
```

---

##  로그 레벨
```
TRACE > DEBUG > INFO > WARN > ERROR
```
```properties
# application.properties
logging.level.root=info                        # 전체 기본 레벨
logging.level.hello.springmvc=debug           # 특정 패키지 레벨
```

> 개발 서버 → debug / 운영 서버 → info

---

##  올바른 로그 사용법
```java
//  잘못된 방법 - 로그 레벨이 info여도 문자 더하기 연산이 실행됨
log.debug("data=" + data);

//  올바른 방법 - 로그 레벨이 info면 아무 연산도 발생하지 않음
log.debug("data={}", data);
```

## 최종 코드
```java
@Slf4j
@RestController
public class LogTestController {

    @RequestMapping("/log-test")
    public String logTest() {
        String name = "Spring";

        log.trace("trace log={}", name);
        log.debug("debug log={}", name);
        log.info(" info log={}", name);
        log.warn(" warn log={}", name);
        log.error("error log={}", name);

        return "ok";
    }
}
```

## 느낀 점
## 느낀 점
- log에 대해 들어봤지만 실제로는 한 번도 쓴 적이 없었다. 근데 성능면이나, 분석을 해야 할 때는 System.out.println() 보다 log가 몇 배는 더 빠르고, 메모리도 덜 써서 좋다는 걸 알게 되었다.
- 로그 레벨(TRACE > DEBUG > INFO > WARN > ERROR)을 배워서 개발 서버와 운영 서버에서 다르게 설정할 수 있다는 게 실용적이었다.
- log.debug("data=" + data) 처럼 + 연산을 쓰면 로그 레벨이 info여도 연산이 실행된다는 걸 알게 되었다. 앞으로는 무조건 log.debug("data={}", data) 방식으로 써야겠다.
