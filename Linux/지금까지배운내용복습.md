## 사용자 관리

### useradd / adduser
| 옵션 | 설명 | 예시 |
|------|------|------|
| `-m` | 홈 디렉토리 생성 | `useradd -m user1` |
| `-d` | 홈 디렉토리 경로 지정 | `useradd -d /home/user1 user1` |
| `-s` | 기본 쉘 지정 | `useradd -s /bin/bash user1` |
| `-G` | 보조 그룹 지정 | `useradd -G sudo user1` |
| `-u` | UID 직접 지정 | `useradd -u 1001 user1` |

> adduser = useradd의 대화형 버전 (비밀번호, 이름 등 물어봄)

### usermod
| 옵션 | 설명 | 예시 |
|------|------|------|
| `-l` | 로그인 이름 변경 | `usermod -l new old` |
| `-d` | 홈 디렉토리 변경 | `usermod -d /home/new user1` |
| `-G` | 보조 그룹 변경 | `usermod -G sudo user1` |
| `-aG` | 보조 그룹 추가 | `usermod -aG docker user1` |
| `-s` | 쉘 변경 | `usermod -s /bin/bash user1` |
| `-L` | 계정 잠금 | `usermod -L user1` |
| `-U` | 계정 잠금 해제 | `usermod -U user1` |

### passwd
| 옵션 | 설명 | 예시 |
|------|------|------|
| `-l` | 계정 잠금 | `passwd -l user1` |
| `-u` | 계정 잠금 해제 | `passwd -u user1` |
| `-d` | 비밀번호 삭제 | `passwd -d user1` |
| `-e` | 다음 로그인 시 비밀번호 변경 강제 | `passwd -e user1` |

### groupadd / groupmod
| 명령어 | 옵션 | 설명 | 예시 |
|------|------|------|------|
| `groupadd` | `-g` | GID 직접 지정 | `groupadd -g 1001 grp1` |
| `groupmod` | `-n` | 그룹 이름 변경 | `groupmod -n new old` |
| `groupmod` | `-g` | GID 변경 | `groupmod -g 1002 grp1` |
| `groupdel` | - | 그룹 삭제 | `groupdel grp1` |

### 주요 파일
| 파일 | 설명 |
|------|------|
| `/etc/passwd` | 사용자 계정 정보 (ID, UID, GID, 홈디렉토리, 쉘) |
| `/etc/shadow` | 암호화된 비밀번호 저장 (passwd보다 보안 강화) |
| `/etc/group` | 그룹 정보 저장 |

## 파일 소유권 & 허가권

### chown vs chmod
| 명령어 | 역할 | 예시 |
|------|------|------|
| `chown` | 파일 소유자 변경 | `chown user:group 파일명` |
| `chmod` | 파일 권한 변경 | `chmod 755 파일명` |

### 권한 구조
```
rwxrwxrwx
│││││││││
│││││││└└└ others (다른 사용자)
│││││└└└── group (그룹)
│││└└└──── owner (소유자)
```
| 권한 | 숫자 |
|------|------|
| r (읽기) | 4 |
| w (쓰기) | 2 |
| x (실행) | 1 |

> `chmod 755` → 소유자 rwx(7), 그룹 rx(5), others rx(5)

### 특수 권한
| 권한 | 설명 | 설정 |
|------|------|------|
| SetUID | 실행 시 소유자 권한으로 실행 | `chmod 4755 파일명` |
| SetGID | 실행 시 그룹 권한으로 실행 | `chmod 2755 파일명` |
| Sticky Bit | 본인 파일만 삭제 가능 (공유 디렉토리) | `chmod 1755 디렉토리` |

> /tmp 디렉토리가 대표적인 Sticky Bit 적용 예시

### umask - 기본 권한 설정
```bash
umask        # 현재 umask 값 확인
umask 022    # umask 설정

# 파일 기본권한 666 - umask 022 = 644 (rw-r--r--)
# 디렉토리 기본권한 777 - umask 022 = 755 (rwxr-xr-x)
```

## 하드링크 vs 심볼릭링크

| 구분 | 하드링크 | 심볼릭링크 |
|------|------|------|
| 생성 | `ln 파일명 링크명` | `ln -s 파일명 링크명` |
| 원본 삭제 시 | 데이터 유지 | 링크 깨짐 |
| inode | 원본과 동일 | 별도 inode |
| 다른 파티션 | 불가 | 가능 |

### inode란?
```
파일의 실제 데이터 위치, 권한, 소유자 등 메타데이터를 저장하는 구조체
파일 이름 ≠ 파일 실체 → inode 번호로 실제 데이터를 찾아감
```

## 디스크 관리

### fdisk - 파티션 나누기
```bash
fdisk /dev/sda     # 디스크 파티션 관리
fdisk -l           # 파티션 목록 확인
```
| fdisk 내부 명령 | 설명 |
|------|------|
| `n` | 새 파티션 생성 |
| `d` | 파티션 삭제 |
| `p` | 파티션 목록 출력 |
| `w` | 변경사항 저장 |
| `q` | 저장 없이 종료 |

### mkfs - 파일시스템 생성
```bash
mkfs.ext4 /dev/sda1    # ext4 파일시스템 생성
mkfs.xfs /dev/sda1     # xfs 파일시스템 생성
```

### mount - 마운트
```bash
mount /dev/sda1 /mnt       # 마운트
umount /mnt                # 언마운트
mount -a                   # /etc/fstab 기준 전체 마운트
```

### /etc/fstab - 자동 마운트 설정
```bash
# 부팅 시 자동으로 마운트할 디스크 설정
# 장치명  마운트포인트  파일시스템  옵션  dump  pass
/dev/sda1  /home  ext4  defaults  0  2
```
> fstab 잘못 설정하면 부팅 안 될 수 있으니 주의

### LVM (논리 볼륨 관리자)
```
물리 디스크(PV) → 볼륨 그룹(VG) → 논리 볼륨(LV)
장점: 용량을 유연하게 늘리고 줄일 수 있음
```
| 명령어 | 설명 |
|------|------|
| `pvcreate /dev/sdb` | 물리 볼륨 생성 |
| `vgcreate vg1 /dev/sdb` | 볼륨 그룹 생성 |
| `lvcreate -L 10G -n lv1 vg1` | 논리 볼륨 생성 |
| `lvextend -L +5G /dev/vg1/lv1` | 논리 볼륨 확장 |
| `lvremove /dev/vg1/lv1` | 논리 볼륨 삭제 |

### df / du - 디스크 사용량 확인
| 명령어 | 설명 | 예시 |
|------|------|------|
| `df -h` | 전체 디스크 사용량 확인 | `df -h` |
| `du -sh` | 특정 디렉토리 용량 확인 | `du -sh /home` |

## 쿼터 (Quota) - 디스크 사용량 제한

특정 사용자나 그룹이 사용할 수 있는 디스크 용량을 제한하는 기능

### 쿼터 설정 순서
```
1. /etc/fstab 에 usrquota, grpquota 옵션 추가
2. 재마운트
3. quota DB 생성
4. 쿼터 설정
5. 쿼터 활성화
```

### 주요 명령어
| 명령어 | 설명 | 예시 |
|------|------|------|
| `quotacheck` | 쿼터 DB 생성 및 검사 | `quotacheck -ugm /home` |
| `quotaon` | 쿼터 활성화 | `quotaon /home` |
| `quotaoff` | 쿼터 비활성화 | `quotaoff /home` |
| `edquota` | 사용자 쿼터 설정 | `edquota -u user1` |
| `repquota` | 쿼터 사용 현황 확인 | `repquota /home` |

### 소프트 리밋 vs 하드 리밋
| 구분 | 설명 |
|------|------|
| 소프트 리밋 | 초과 가능하지만 경고 발생, 유예기간 내 초과 허용 |
| 하드 리밋 | 절대 초과 불가, 즉시 차단 |

## RAID (Redundant Array of Independent Disks)

여러 개의 디스크를 묶어서 하나의 디스크처럼 사용하는 기술
속도 향상 또는 데이터 안정성을 목적으로 사용

### RAID 종류 및 비교
| 구분 | 방식 | 최소 디스크 | 장점 | 단점 |
|------|------|------|------|------|
| RAID 0 | 스트라이핑 | 2개 | 속도 빠름 | 1개 고장 시 전체 데이터 손실 |
| RAID 1 | 미러링 | 2개 | 1개 고장해도 데이터 유지 | 용량 절반만 사용 |
| RAID 5 | 스트라이핑 + 패리티 | 3개 | 속도 + 안정성 균형 | 1개 고장까지만 복구 |
| RAID 6 | 스트라이핑 + 패리티 2개 | 4개 | 2개 고장까지 복구 | 쓰기 속도 느림 |
| RAID 10 | RAID 1 + RAID 0 | 4개 | 속도 + 안정성 모두 우수 | 비용 높음 |

### 실무에서 많이 쓰는 RAID
```
개인/소규모  → RAID 1 (안정성 우선)
서버/기업    → RAID 5, RAID 10 (속도 + 안정성)
```

## 느낀 점
- 작년에 배운 내용이지만 복습할 겸 다시 배우니까 개념이 정확히 잡힌 것 같다.
- RAID 개념은 처음 배웠는데 각각의 장단점이 있으니 상황에 맞게 잘 선택해서 사용해야겠다.
- 앞으로 네트워크 개념이 중요해질 것 같다. SSH, 텔넷 관련 내용을 배울 예정이므로 네트워크 개념도 다시 익혀야겠다.
