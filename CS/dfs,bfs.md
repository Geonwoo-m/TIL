# DFS / BFS 개념 정리

## DFS (깊이 우선 탐색, Depth-First Search)

- 한 방향으로 갈 수 있을 때까지 깊게 들어가다가 막히면 되돌아오는 방식
- **Stack** 또는 **재귀함수** 로 구현
- 모든 경로를 탐색해야 할 때 유리
```
그래프 예시
    1
   / \
  2   3
 / \
4   5

DFS 탐색 순서: 1 → 2 → 4 → 5 → 3
```

---

##  BFS (너비 우선 탐색, Breadth-First Search)

- 현재 위치에서 가까운 노드부터 탐색하는 방식
- **Queue** 로 구현
- **최단 경로** 를 찾을 때 유리
```
그래프 예시
    1
   / \
  2   3
 / \
4   5

BFS 탐색 순서: 1 → 2 → 3 → 4 → 5
```

---

## 💡 DFS vs BFS 비교

| 구분 | DFS | BFS |
|------|-----|-----|
| 자료구조 | Stack / 재귀 | Queue |
| 탐색 방식 | 깊게 파고듦 | 가까운 곳부터 |
| 최단 경로 | x | o |
| 사용 예시 | 모든 경우의 수, 백트래킹 | 최단 거리, 레벨 탐색 |

---

## 💡 언제 DFS? 언제 BFS?
```
최단 거리, 최소 횟수 → BFS
모든 경로 탐색, 순열/조합 → DFS
```

---

## 느낀 점
- DFS는 Stack/재귀로 깊게 파고들고, BFS는 Queue로 가까운 곳부터
  탐색한다는 차이가 핵심이었다.
- 최단 경로는 BFS, 모든 경우의 수는 DFS라는 기준을 기억해두고
  문제 유형에 맞게 골라써야겠다.
